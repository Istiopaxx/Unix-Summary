Unix Summary - Chapter 8

Regular Expression

  • A syntactical set or phrase that represents a pattern of text or strings
  • Enables you to represent a varying array of characters with a much smaller set of predefined characters
  • Often include metacharacters
  • Metacharacters : characters that represent another set or group of characters or commands

Meta characters

  • . (dot, period) : a character
  • [] (square brackets) : any one of inside in it
  • * (asterisk, splat) : zero or more characters
  • [^.*] (caret) : do not match any one of inside in it
  • ^.* (caret) : match at the beginning
  • ^.*[.*] : match at the beginning and match inside in it
  • $ (dollar) : match at the end of line
  • \ (backslash) : remove special meaning of the following character
  • ? (question mark) : zero or one character
  • [a-z] : match all occurrences of any single letter

Regular Expression

grep

  • $ print lines that has the string
  • $ grep unix30 /etc/passwd
  • $ gat /etc/passwd | grep unix30
  • $ grep –v // opposite
  • $ grep –i // ignore case

find

  • $ Find files in directory structure
  • $ find / -name core
  • $ find /export –user unix300
  • $ find /export –user unix300
  • $ find /export –user unix300
  • $ find / -name core 2> /dev/null

sort

  • $ sort –t: +1 /etc/group
  • $ head /etc/passwd | sort –r > pass.txt
  • $ -f : ignore case
  • $ -r : reversed order
  • $ -t : set delimiter (field separator)
  • $ -o : set output file

Tee/script

  • Tee : split output (screen and file)
  • – $ ps –ef | tee ps.txt
  • – $ telnet 10.10.54.75 | tee telnet.txt
  • Script : record interactive login session
  • – $ script [-a filename]
  • – $ unix commands
  • – $ …
  • – $ exit // script is ended

  • paste : file merge
  • tr : translate character
  • split : file divide
  • uniq : file compare
  • cmp : file compare
