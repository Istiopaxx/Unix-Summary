Root directory (1)

  • $ / : root directory of the entire file system hierarchy
  • $ /devices : physical device files
  • $ /dev : logical device files
  • $ /etc : host-specific system-wide configuration files
  • /etc is referred to as the etcetera directory, as this directory historically held everything that did not belong elsewhere. Recent interpretations include backronyms such as "Editable Text Configuration" or "Extended Tool Chest"
  • $ /mnt : temporarily mounted filesystems.
  • $ /opt : optional application software packages
  • $ /proc : virtual filesystem providing process and kernel information as files
  • $ /tmp : temporary files. Not preserved between system reboots.

Root directory (2)

  • $ /bin : command binaries (same with /usr/bin)
  • $ /sbin : Non-essential system binaries, e.g., daemons for various network-services
  • $ /usr : used for miscellaneous purpose – /usr/bin, /usr/lib, /usr/include
  • $ /lib : libraries for the binaries in /usr/bin/ and /usr/sbin/ (same with /usr/lib)
  • $ /var : Variable files whose content is expected to continually change during normal operation of the system., such as logs, spool files, and temporary e-mail files.
  • $ /home : Users' home directories(Now user’s home is in /export/home)
  • $ /export : open contains remote file systems

File types

  • $ First character of ‘ls –l’ command
  • d : The entry is a directory
  • b : The entry is a block special file
  • c : The entry is a character special file
  • l : The entry is a symbolic link
  • p : The entry is a first-in, first-out (FIFO) special file (pipe)
  • s : The entry is a local socket
  • - : The entry is an ordinary file
  • $ File hi.txt (unix command) – Display file type – hi.txt : ASCII Text

permission

  • $ rw-r--r--
  • $ User/owner(rw-), group(r--), other(r--)
  • $ R(Read), w(write), x(execute)
  • – File readable : cat possible
  • – File writable : can edit
  • – File executable : can run that file (program)
  • – Directory readable : ls possible
  • – Directory writable : create/delete possible
  • – Directory executable : cd possible

umask

  • $ Defines masking some permissions
  • – When make a new file or a new directory
  • $ Bit 1 masks permission (not give permit)
  • $ umask [value]
  • $ umask // see current value
  • $ umask 022 // set umask as 022

Default file/dir permission

  • $ 777 XOR umask -> DEF_DIR_MASK
  • – default directory permission
  • $ Remove x from DEF_DIR_MASK
  • – default file permission
  • $ Example
  • – If umask is 022
  • – 777 XOR 022 = 755
  • • default directory permission
  • – X remove from 755 = 644
  • • default file permission
